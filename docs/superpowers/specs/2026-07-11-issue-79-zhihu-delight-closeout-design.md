# Issue #79 知乎惊喜卡收尾设计

## 背景

Issue #79 的既有实现已经完成以下内容：

- 知乎标题缺失时，从正文首句生成标题，并在无正文时使用可读占位文案；
- 从知乎多种响应字段提取封面；
- 推荐网格的文字卡支持封面模糊背景；
- 推荐网格和惊喜卡已展示可用的点赞、评论、收藏等互动指标；
- 桌面惊喜卡不再直接显示存量 `answer_<id>` 标题。

剩余缺口集中在桌面 Web 的惊喜推荐卡：后端已经下发 `body_text`，但
`normalizeDelight()` 没有把它保留到前端状态，卡片也没有正文预览和展开交互。
无封面时左侧媒体区仍是空泛的渐变占位，未充分利用文字内容。

## 目标

在不重构三端推荐 UI 的前提下，完成桌面 Web 惊喜推荐卡的文字内容体验：

1. 对知乎等文字优先内容展示约 100 字的正文预览；
2. 正文较长时提供明确、可键盘操作的展开/收起交互；
3. 无封面时用正文预览填充左侧媒体区；有封面时沿用现有整卡模糊背景和封面；
4. 保持现有标题兜底、统计信息、队列切换、聊天和反馈行为不变。

## 范围

本次只修改桌面 Web `/web` 的惊喜推荐卡及其直接测试、文档。移动 Web 和浏览器
插件已经有各自独立的卡片布局，不在本次做三端视觉重构。推荐网格现有的文字卡
行为也保持不变。

## 交互与视觉设计

### 正文预览

- `normalizeDelight()` 解码并保留 `body_text`。
- 仅当 `body_text` 非空时渲染正文区域。
- 默认展示最多 5 行，目标信息量约等于 80～120 个中文字符；截断由 CSS 行数控制，
  DOM 中保留完整正文，避免前端按字节破坏 Unicode 文本。
- 正文区域与“推荐原因”分开：正文回答“内容在讲什么”，推荐原因回答“为什么推荐给你”。

### 展开与收起

- 仅在正文实际溢出时显示“展开正文”；展开后显示完整正文并切换为“收起正文”。
- 按钮使用原生 `button`，同步 `aria-expanded`，正文容器使用稳定 ID 和
  `aria-controls` 关联。
- 切换惊喜候选时恢复折叠态，避免上一条的 UI 状态泄漏到下一条。
- 展开正文不触发打开内容、队列切换、反馈或聊天行为。

### 封面与无封面状态

- 有有效封面：保持现有左侧封面和整卡模糊背景，正文出现在右侧内容区。
- 无封面且有正文：左侧媒体区改为文字预览卡，显示正文开头和来源平台徽章；
  使用现有来源渐变、半透明表层和柔和模糊，不伪造图片。
- 无封面且无正文：保留现有通用渐变占位和平台徽章。
- 图片加载失败时退化到与“无封面且有正文”相同的文字预览状态。

## 数据流

`/api/delight/pending-batch` 和实时 `delight.candidate` 事件已经携带
`content_type` 与 `body_text`，因此不新增 API 字段或数据库迁移：

```text
content_cache.body_text
  -> pending-batch / delight.candidate
  -> normalizeDelight().body_text
  -> setActiveDelight()
  -> 正文预览 + 无封面文字媒体区
```

所有输出继续走 `textContent` 或既有 HTML 转义函数，不把正文当 HTML 注入。

## 状态与错误处理

- 空正文：不渲染正文区域和展开按钮。
- 图片 URL 无效或加载失败：移除失败图片并渲染文字媒体区；没有正文时显示通用占位。
- 候选为空：沿用“暂无惊喜队列”，同时清空正文和展开控件。
- 新候选实时到达但用户正在聊天：沿用现有 `delightUserEngaged()` 守卫，不强制重绘当前卡。
- 自动轮播和手动上一条/下一条切换后，重新检测新正文是否溢出。

## 测试与验收

### 自动测试

- 前端静态/Node 合约测试覆盖 `body_text` 归一化、正文 DOM、无封面文字媒体区、
  `aria-expanded` 与图片失败降级。
- 现有知乎标题、封面、互动指标和惊喜卡 API 测试继续通过。
- 运行桌面前端 JavaScript 语法检查、Ruff、MyPy 和相关 Pytest；最终运行仓库全量测试。

### 浏览器验收

用本地桌面 Web 验证四种状态：

1. 知乎回答有封面、有长正文：封面保留，右侧正文可展开/收起；
2. 知乎回答无封面、有长正文：左侧显示文字媒体区，右侧正文可展开/收起；
3. 正文较短：完整显示且不出现展开按钮；
4. 无正文：不出现空白正文区域，原有卡片行为不变。

同时验证鼠标、键盘、队列切换、图片加载失败、聊天输入和反馈按钮不互相干扰。

- 不改变知乎采集长度、推荐排序、惊喜阈值或互动指标定义；
- 不让正文展开直接替代“去看看”打开原内容。
