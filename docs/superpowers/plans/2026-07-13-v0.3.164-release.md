# OpenBiliClaw v0.3.164 Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish OpenBiliClaw v0.3.164 across source, Docker, extension, desktop, aggregate release, and Chrome Web Store review channels.

**Architecture:** Use `scripts/release.py` as the mechanical version source, then update the release-facing changelog and bilingual README highlights. Merge one tested release commit to `main`, push the four tag-triggered channels one at a time, wait for every workflow, and finally replace the pending Chrome Web Store submission with v0.3.164.

**Tech Stack:** Python release helper, uv lockfile, Node/WebExtension packaging, GitHub Actions, GitHub Releases, GHCR, Chrome Web Store API v2.

## Global Constraints

- Backend, extension, desktop, aggregate, and Chrome Web Store package versions are exactly `0.3.164`.
- Push tags individually in this order: `backend-v0.3.164`, `extension-v0.3.164`, `desktop-v0.3.164`, `openbiliclaw-v0.3.164`.
- Chrome Web Store dispatch uses `publish=true`, `staged=false`, and `replace_pending=true` because v0.3.163 is already pending review.
- Firefox AMO signing remains disabled while repository variable `FIREFOX_SIGNING_ENABLED=false`; do not claim an AMO submission or signed XPI.
- Do not include generated zip, XPI, DMG, EXE, `dist/`, or local credentials in Git.
